MyFlix App: Case Study

Case Study Overview

MyFlix is a info-movie app, created for people who loves movies and want to know more about cinema like the All Movie Guide. As a user-friendly app, MyFlix is build to provide users with the best possible experience where they can access information about different movies, actors, actresses, directors, genres and add movie(s)to their list of favourite movies.

Purpose & Context

MyFlix was a personal project I built as part of my web development program at CareerFoundry, to domonstrate my mastery of full-stack JavaScript development.

The goal/purpose of the project was to teach and prepear students/developers like me to gain a complete knowldge, understanding and experience in Front-end and Back-end development as a Full-Stack Developer, using some of the most common mordern tech stacks, MERN, (Express, React, JavaScript, CSS, HTML and Node.js).

Objective & Methodology

The aim of the project was to have an ambitious full-stack project I can add to my professional portfolio. The problem I wanted to solve is to build the complete server-side and client-side for the application from scratch.

The Methodology used to build the project include the following:

A Server-side method/programming which involes the used of Node.js and Express for securelly accessing information over the internet - which also interacts with MongoDB, a public database framework. To access the API, the CRUD (create, read, update and delete), including the post method was implemented with the result given in a JSON format.

Client-Side: In the client-side, React (a JavaScript library) and React-Redux were utilize to connect the application with the sever (i.e., the back-end of the app). This enables users to interact with the app by creating an account or logging in with their credentials to access and/or navigate between the app main view where they can find all movies along with their information.

Server-Side Approach

For the Back-end of the app, in order to store movies data and users information, a server-side with RESTful API (interface to exchange information securely over the internet) was built with Node.js and Express, which interacts with a database in MongoDB. To access the API the CRUD (create, read, update and delete), including the post method was implemented witht the result given in a JSON format (i.e., a text-based format for Javascript object syntax). Postman was also utilized as a recommanded tool to test each executed request such as the create, read, update, delete and post methods

An important step was to include user authentication and authorization logic in the form of basic HTTP and JWT (JSON Web Token), a standard of internet tool of which upon user successful registration, based on their given credentials, generates an encrypted token to prevent any security issues that may occur.


Client-Side Approach

For better/best user experience, the app web page was build with React (a modern Javascript library) and React-Redux (a state manager to guarantee consistent behaviour across server and client logic). Tech stack like Bootstrap, a (Javascript styling library) with pre-definded built-in components was also used for best pratice and responsive design pueposes for the app.

Upon registration or login on the welcome page, users can navigate between the main view of the app, where a movie list and a search tool is provided to quickly find specific movies; a movie details view that shows more information about the movie like Director info, Movies Genre, and Summary each with a separate view providing users with important information about each movies. In the app, user can also edit their profile by either deleting their account, update current profile data like chaning the password, email address date of birth, or their username. Users can also add and remove movie(s) from their list of favourite movies.


Lessons learned

I learned new languages and libraries like react-redux, react-router, mongoose and mongo DB My understanding of how websites function from client-side, to server-side and then database was strongly enhanced during this project.

Another important lesson learned why working on this project/app was saving more time to reach the maximum expectations of the course schedule even though there were some health issues and concerned.

Challenges Faced

Working on this app/project was a great exprience for me especally using JavaScript, Node.js Bootstrap and React. There were few challeges mostly on the Back-end or server side of the application. As a new programer, I had to learn many new syntax of many libraries. For instance, it was my first time learning react-redux and moongoose syntax for business logic for applications. I also had some struggles with deep debugging at certain point in the project.

However, with determination, persistence, some research, and hlep from my Mentor and Tutor, I was able to addressed each problems by followin all steps and procedures such as using the the terminal, reading the documentations for new frameworks and libraries, doing online research for articles or community answers to the specific issues I was facing, and using the google console for debugging functions and apsects of the app that was generating errors.

Summary

My original intention for the app as it comprieses server and client-side implementation, was to build and design a website that's both fun and delightful to use without any complications or distracting, and/or negatively effecting the targeted demographic of uers and professionals who interact with it. My biggest challenge was understanding the multiples syntax for the different libraries and frameworks used for the application; especally in the back-end side of the development process. As a critical thinker, I wanted to guide the project in a direction where I would be able to inject personality into it and push my creativity at high level.

As a result, I am very proud of the app quirky look and feel I have build. All thanks to CareerFoundry who not only provids execellent programs but also an amazing professionals and well experience team of Mentors and Tutors who supports thier students in the best way possible.

Moving forward, if I'm to do things differently, I would like to approach more projects with a visual brainstorm and ideate on what type of personality I'd like to give an app and its design. A fun question to ask is, if I was a user interacting with myFlix Appp or any app for that matter, what would I like/hope its design look and functionality to be? Would it be functional and user-friendly? Such questions could only be answer based on the developer exprience and professional workflow/ethic.
